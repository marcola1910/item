Protótipo de tecnologia

Este site foi criado como prova de conceito de uma arquitetura que separa totalmente a camada de dados da camada de negócio. A grande vantagem é a de oferecer um desenho mais flexível por poder aplicar os princípios da orientação a objetos, além de transparência com o banco de dados tornando a lógica independente do motor de base de dados. Podendo funcionar até mesmo puramente na memória sem reescrever uma linha de código.

Isso também auxilia nos testes funcionais, por estar fora do banco de dados podemos armar set de dados de entrada mais personalizados e mais facilmente.
Os princípios que foram seguidos são os do Domain-Driven-Development e os padrões de desenho do gang of four. Além de utilizar alguns padrões de Martin Fowler descritos no livro : Patterns of Enterprise Application Architecture facilitando a injeção de dependências permitindo IoC (inversão de controle) Tais como:

  • Root Object
  • Service Locator
  • Repository
  • Repository Locator

Transações

Por estar trabalhando neste momento com um motor relacional MySQL e ACID, utilizamos o RootObject para que faça persistência por alcance dentro de uma transação atômica. É dizer, que cada serviço do sistema abre uma transação atômica, caso surja uma exceção o sistema fará rollback.

Concorrência

Por meio da integração Hibernate e Spring podemos controlar o fluxo de transações e configurar o regime de concorrência. Neste caso está “O primeiro ganha”. Caso outro usuário em paralelo tome o dado “sujo” o sistema se encarga de validar e mostrar a devida mensagem ao ususario.
